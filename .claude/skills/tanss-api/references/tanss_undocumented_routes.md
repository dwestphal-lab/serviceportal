# TANSS API — Undokumentierte Routen

> Diese Datei wird automatisch und manuell erweitert.
> Hier werden API-Routen dokumentiert, die NICHT in der offiziellen combined.yaml stehen,
> aber in der Praxis entdeckt oder von TANSS-Support mitgeteilt wurden.

## Eintragungsformat

Neue Routen in folgendem Format eintragen:

```markdown
### METHODE /pfad/zur/route
- **Entdeckt:** YYYY-MM-DD
- **Quelle:** (z.B. "TANSS Support", "Reverse Engineering", "Kundenprojekt XY")
- **Auth:** (Token-Typ, z.B. "Employee Login", "ERP Token")
- **Request Body:**
  ```json
  { "field": "type — Beschreibung" }
  ```
- **Response:**
  ```json
  { "field": "type — Beschreibung" }
  ```
- **Notizen:** Zusätzliche Informationen
```
